# Liquid Attention Multimodal CoT 

This repository implements a local, CPU-friendly PyTorch baseline for:
- Liquid Attention with continuous-time dynamics (RK4)
- Cross-step attention state propagation
- Self-validation early stopping
- Comparison and ablation experiment suites

## Project Structure

- `src/`: core library code
- `configs/`: dataset and experiment configs
- `scripts/`: CLI entrypoints
- `data/raw/`: place local raw datasets here
- `data/processed/`: normalized JSONL generated by `prepare_data.py`
- `outputs/`: checkpoints, logs, and report tables
- `tests/`: unit/integration tests (unittest)

## Quick Start

1. Prepare data (local-only):

```bash
python3 scripts/prepare_data.py --config configs/datasets/local_v1.yaml
```

2. Run full experiment matrix (3 datasets x 9 unique training configs x 1 seed):

```bash
python3 scripts/run_experiments.py --config configs/base.yaml --matrix configs/experiments/experiment_matrix.yaml
```

3. Collect final tables and figures:

```bash
python3 scripts/collect_results.py --runs-dir outputs/runs --output-dir outputs/reports
```

## One-command workflow

```bash
bash scripts/run_all.sh
```

## Notes

- Default device policy is CPU-first with optional MPS auto-detection.
- No online downloads are required. The pipeline reads local files under `data/raw/<dataset>`.
- If raw data is missing, `prepare_data.py` writes `outputs/reports/missing_data_report.json`.
